# NOTES

## FONTS FOR VS CODE

1. Check out Fira Code font, install it [link to download & install](https://github.com/tonsky/FiraCode/wiki)
2. Go to preferences, then settings
3. Search Font and change
4. While in the same section, enable font ligatures

## PRETTIER

1. For formatting js
2. Install prettier `bash npm install -D prettier`
3. In the package.json, add the following command
   ```json
       "format": "prettier \"src/**/*.{js,html}\" --write"
   ```
   This runs on all files and directories in the /src with the js and html extension
4. To make prettier run everytime you save, install the vscode plugin for prettier. Go to settings and turn on Editor: format On save. After than, search 'require config' and scroll down to the prettier section and make sure it is checked. Finally create a config file at the root directory and save it as '_.prettierrc_'
5. In the .prettierrc, add the following json: it's also fine using an empty object for the defaults.
   ```json
   {
     "singleQuote": false,
     "useTabs": false,
     "tabWidth": 4
   }
   ```

## ESLINT

1.  Prettier concerned with format primarily, doesn't care about code. ESlint is concerned with style. eslint-config-prettier makes sure prettier and eslint work well together
2.  `bash npm install -D eslint eslint-config-prettier`
3.  At the root create '_.eslintrc.json_'
4.  Inside the file, add the following
    ```json
    {
      "extends": ["eslint:recommended", "prettier", "prettier/react"],
      "plugins": [],
      "parserOptions": {
        "ecmaVersion": 2018,
        "sourceType": "module",
        "ecmaFeatures": {
          "jsx": true
        }
      },
      "env": {
        "es6": true,
        "browser": true,
        "node": true
      }
    }
    ```
5.  In the package.json, add the following command
    ```json
        "lint": "eslint \"src/**/*.{js,jsx}\" --quiet",
    ```
6.  Make sure the eslint plugin is installed and that eslint is run on save rather than on type

## GITIGNORE

1. create file in root directory '_.gitignore_'
2. Add the following dirs and files to .gitignore
   - node_modules/
   - .DS_Store
   - .cache/
   - dist/
   - coverage/

   - .env

## PARCEL

1. `bash npm install -D parcel-bundler`
2. In package.json, add the following script. Point parcel to the index.html and it figures out everything else
   ```json
       "dev": "parcel src/index.html"
   ```

## REACT AND REACTDOM

1.  `bash npm i react react-dom`
2.  Make sure npm intellisense is installed in vscode
3.  Add the following at App.js:
    ```javascript
    import React from "react"
    import { render } from "react-dom"
    ```
4.  Finally add the following. babel-eslint allows for eslint to be augmented by babel, eslint-plugin-import gives new rules for import/export, a11y for accessibility, eslint-plugin-react for additional react rules needed:
    ```bash
        npm install -D babel-eslint eslint-plugin-import eslint-plugin-jsx-a11y eslint-plugin-react
    ```
5.  Configure eslint for react as follows. Prettier and prettier/react have to come last:
    ```json
    {
      "extends": [
        "eslint:recommended",
        "plugin:import/errors",
        "plugin:react/recommended",
        "plugin:jsx-a11y/recommended",
        "prettier",
        "prettier/react"
      ],
      "rules": {
        "react/prop-types": 0,
        "no-console": 1
      },
      "plugins": ["react", "import", "jsx-a11y"],
      "parserOptions": {
        "ecmaVersion": 2018,
        "sourceType": "module",
        "ecmaFeatures": {
          "jsx": true
        }
      },
      "env": {
        "es6": true,
        "browser": true,
        "node": true
      },
      "settings": {
        "react": {
          "version": "detect"
        }
      }
    }
    ```
6.  Installing the following ESLint rules for hooks, eg dont call them inside conditional expressions:

        ```bash
            npm install -D eslint-plugin-react-hooks
        ```

    and inside the rules section of .eslintrc.json, add the following at the end:

    ```json
        "react-hooks/rules-of-hooks": 2,
        "react-hooks/exhaustive-deps": 1
    ```

    Furthermore, in the plugin api, add the following:

    ```json
        "plugins": ["react", "import", "jsx-a11y", "react-hooks"],
    ```
